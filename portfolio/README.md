# ClimbQuest - CPT208 Coursework Portfolio

## 1. Project Overview

This portfolio is a static multi-page website (`HTML + CSS`) created for the CPT208 Human-Centered Computing coursework.

Theme: **ClimbQuest** (indoor climbing scenario)

Main pages:

- `index.html`: Project overview and core features
- `pages/motivation.html`: Project motivation
- `pages/research.html`: Research process and evidence
- `pages/users.html`: User needs and personas
- `pages/ideation.html`: Ideation and comparison
- `pages/prototype.html`: Prototype and system design
- `pages/evaluation.html`: Usability evaluation and reflection
- `pages/references.html`: References
- `pages/team.html`: Team members and responsibilities

## 2. Requirements Covered

This portfolio demonstrates:

- A complete human-centered design process from problem framing to evaluation
- Core climbing-app feature planning, including:
  - Route exploration and challenge discovery
  - Progress tracking and achievement feedback
  - Coach/community feedback support
- A direct web-based submission format for instructors
- A shared team repository for collaboration

## 4. Team Workflow (GitHub <-> Local)

### 4.1 Clone for the first time

```bash
git clone https://github.com/XYMMM000/CPT208-coursework.git
cd CPT208-coursework
```

### 4.2 Pull latest changes before work

```bash
git pull origin main
```

If your default branch is not `main`, replace it with your actual branch name.

### 4.3 Push local updates to GitHub

```bash
git status
git add .
git commit -m "Update: describe your changes"
git push origin main
```

## 5. Local Preview

Open `portfolio/index.html` directly in your browser, or use VS Code Live Server.

## 6. If the Live URL does not open

Check these items in GitHub:

1. `Settings -> Pages` is enabled
2. Source branch is correct (usually `main`)
3. Folder is correct (usually `/root`)
4. Wait 1-3 minutes after a fresh push (GitHub Pages deploy delay)
